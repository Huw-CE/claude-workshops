Open Claude
MCP Tools · 20 min

Give Claude
hands.

MCP stands for Model Context Protocol. It's how Claude connects to your other tools — not by copy-pasting between tabs, but by actually reaching into apps and doing things. Think of it as giving Claude a browser, a calendar, a Gmail account, and a Figma file all at once.

Tools Worth Connecting

Connect once. Available everywhere.

You connect MCP tools once in settings, and they're available in every conversation. Here's what each one does for Charles Elena specifically.

Tool What it does for CE
Figma Reads your design files, extracts tokens, writes dev specs, audits for accessibility, and bridges design to code. The most valuable MCP in our stack — and what we're demoing today.
Google Drive Finds files, summarises briefs, drafts deliverables from existing docs, organises folders. Saves 20 minutes of hunting per day at minimum.
Gmail Summarises long threads, drafts responses in your tone, flags things that need action. Never stare at a 47-email thread again — Claude reads it and tells you what matters.
Google Calendar Creates meeting agendas, preps you for client calls with context from Drive and Gmail, schedules follow-ups automatically.
Zapier Connects everything to everything. Automate handoffs between tools that don't natively talk to each other — client approval to project board, brief received to kickoff doc created.

How to connect an MCP tool

01

Open Settings

In Claude Desktop, click your profile → Settings → Integrations tab. This is where all MCP connections live.

02

Select the tool

Find Figma, Google Drive, Gmail, or Zapier in the list. Click "Connect" and follow the OAuth flow for your account.

03

Use it in any chat

That's it. In any conversation, Claude can now access that tool. Mention the file, folder, or action — Claude finds it.

Live Demo · 10 min + Team Activity

Claude meets Infosys.

Here's our scenario: Infosys has briefed Charles Elena to design a consumer-facing AI experience for their stand at a major tech expo. Attendees walk up and generate custom images and short videos using a personalised AI generator — think photorealistic portraits blended with Infosys brand assets. We're designing the UI for that generator. Figma file is open. Claude is connected via MCP. Let's go.

1
"Look at this Figma file and extract the full design token set — all colours, typography styles, spacing values, and component names. Format it as a structured developer handoff document."
2
"Based on the UI screens in this file, write a complete accessibility audit. Flag any contrast issues, missing labels, or interaction patterns that won't meet WCAG 2.1 AA standards. Be specific — call out the exact components."
3
"We're designing for a public event environment — large touchscreen kiosks, low lighting, users who've never seen this interface before. Critique this UI for that context. What would you change and why?"

Your turn.

Connect the Figma MCP in your Claude settings (Settings → Integrations → Figma). Point it at any file you're currently working on. Ask Claude one question — a critique, a spec request, a content suggestion, or an accessibility check. You've got five minutes. Share the most surprising result with the group.
